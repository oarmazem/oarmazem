Dissecando o Primeiro HTML


  <!DOCTYPE html>
  <html lang="pt-br">

  <head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Armazém</title>
  </head>

  <body>
    <h1>Armazém Café e Relíquias</h1>
    <p>Neste espaço está em construção o futuro site do Armazém Café e Relíguias.</p>
    <p>Parada obrigatória de todo visitante em Cunha</p>
  </body>

  </html>

Acima você está vendo exatamente o código HTML da página do seu armazém, do jeito que ela está agora.

Este código é o que faz seu navegador exibir:

Vamos dissecar este código…

Logo no início vemos <!DOCTYPE html> mas isso, ao contrário de todo o resto que vemos entre parênteses angulares, não é uma tag!

Esta é a 1ª coisa em qualquer documento HTML e serve para o navegador saber que versão de HTML a página está usando. (Neste caso estamos utilizando a última versão do HTML, o HTML5 ).

Logo em seguida vemos a tag <html> e esta indica o início do documento. Fecha com a tag de fechamento </html> (Note que a presença de uma barra no interior da tag é o que denota ser a tag de fechamento)

<html> e seu respectivo fechamento </html> delimitam para o navegador o próprio documento HTML.

No escopo da tag <html> você vê as já referidas tags <head> e <body> que respectivamente delimitam o cabeçalho e o corpo da página.

Examinemos o que há dentro de cada uma delas…

Dentro do HEAD (no seu escopo limitado por <head> e </head>) você vê 3 tags <meta> e uma tag <title>.


  <head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Armazém</title>
  </head>

Não é preciso agora entender nem decorar a função delas. Até porque os editores que a gente utiliza para escrever código HTML já inserem automaticamente estas tags obrigatórias.

Mas vou falar brevemente sobre elas apenas pra você começar a compreender a estrutura de um documento HTML.

Como dito antes, o <head> é a seção do documento HTML onde inserimos as configurações. A tag <meta> é de "meta-dado", ou seja, fornece dados sobre os dados.

<meta charset="UTF-8"> Esta é importante e diz ao navegador como os caracteres estão codificados. Como você deve imaginar, em um computador todos os caracteres são representados por valores númericos. São codificados por números. UTF-8 é uma das codificações que existem e esta tag está informando ao navegador que os caracteres do arquivo HTML estão representados em UTF-8. O que obviamente o seu browser precisa saber para processar o texto.

<meta http-equiv="X-UA-Compatible" content="IE=edge"> Esta é para forçar a compatibilidade com certos navegadores da Microsoft. Não é preciso entender, apenas note como estas tags <meta> são inseridas no HEAD para fornecer este tipo de informação para o navegador.

<meta name="viewport" content="width=device-width, initial-scale=1.0"> Esta também é um pouco difícil explicar agora. Mas é necessária para criar sites "responsivos", que alteram seu layout e comportamento de acordo com o dispositivo onde está sendo visualizado.

<title>Armazém</title> Por fim esta tag é a que está dizendo ao navegador qual título ele deve colocar na aba onde irá abrir esta página.

Bem, estas são as tags dentro do escopo da tag <head> deste documento. Agora vejamos o que há no <body>


  <body>
    <h1>Armazém Café e Relíquias</h1>
    <p>Neste espaço está em construção o futuro site do Armazém Café e Relíguias.</p>
    <p>Parada obrigatória de todo visitante em Cunha</p>
  </body>

Está vendo? É o conteúdo propriamente dito da página.

As tags <h1></h1> delimitam um título. E as tags <p></p> delimitam um parágrafo.

Portanto você vê na página um título e dois parágrafos. Agora experimente editar os códigos exibidos nesta página (sim, são editáveis!) e clique no botão Edite e experimente! para ver o resultado.

Altere o título, os parágrafos, adicione novos parágrafos e títulos. Ou delete alguma tag se preferir. Mexa no código à vontade para começar a se familiarizar e observe o resultado das suas edições.

Neste site a maioria dos exemplos serão editáveis e você deve experimentar o que quiser. É a melhor maneira de tirar qualquer dúvida.
