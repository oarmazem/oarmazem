Formatação de Código HTML

Neste artigo vamos falar sobre algumas particularidades ao escrever e editar código HTML.

Dê uma boa olhada neste código. Você nota que algumas linhas iniciam mais à direita enquanto outras têm um recuo maior, estão posicionadas mais próximas da margem esquerda do documento. Mas qual o motivo desse desalinhamento?


  <!DOCTYPE html>

  <html lang="pt-br">
    
  <head> 
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Um Exemplo</title>
  </head>

  <body> 
    <h1>Armazém Café e Relíquias</h1> 

    <p>Neste espaço está em construção o futuro site do Armazém Café e Relíquias.</p> 
    <p>Parada obrigatória de todo visitante em Cunha.</p>

    <p lang="en-us">Here is under construction the future site of Armazém Café e Relíquias.</p>
    <p lang="en-us">An obligatory stop for every visitor in Cunha.</p>
  </body>

  </html>

Isto se chama indentação e é um recurso útil para tornar mais legível o código de uma linguagem de computador.

Indentar o código permite ao leitor humano enxergar com mais facilidade a estrutura do código. O quê está "dentro" do quê.

Assim, no código acima, as tags no escopo do <head> estão avançadas dois espaços para a direita em relação ás tags <head> e </head>, tornando mais visível que elas próprias pertencem ao escopo do HEAD.

Devemos inserir espaçamentos horizontais e verticais em um código fonte para ajudar a evidenciar a estrutura do código e torna-lo mais legível. Embora isto não faça diferença para o "browser", que na verdade não toma conhecimento de espaçamentos horizontais ou verticais.

Observe uma versão sem espaçamentos do mesmo código, note como se torna um pouco mais difícil de ler:


  <!DOCTYPE html>
  <html lang="pt-br">
  <head> 
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Um Exemplo</title>
  </head>
  <body> 
  <h1>Armazém Café e Relíquias</h1> 
  <p>Neste espaço está em construção o futuro site do Armazém Café e Relíquias.</p> 
  <p>Parada obrigatória de todo visitante em Cunha.</p>
  <p lang="en-us">Here is under construction the future site of Armazém Café e Relíquias.</p>
  <p lang="en-us">An obligatory stop for every visitor in Cunha.</p>
  </body>
  </html>

No entanto o navegador leria ambos os códigos da mesma forma e exibiria exatamente a mesma página para ambos.

Há algo que devemos saber ao escrever nossos próprios códigos em HTML: ele não toma conhecimento de quebras de linha e nem de espaços em branco entre as palavras!

Observe os 3 parágrafos no BODY (tudo entre as tags <body> e </body>) do código HTML abaixo:


  <!DOCTYPE html>
  
  <html lang="pt-br">
    
  <head> 
    <meta charset="UTF-8">
    <title>Exemplificando Espaçamentos</title>
  </head>
  
  <body> 
    <h1>Os 3 parágrafos abaixo são exibidos da mesma forma:</h1> 

    <p>Vovô viu a uva.</p>
    
    <p>Vovô          viu          a           uva.</p>

    <p>
    Vovô 
    viu 
    a 
    uva.
    </p>
  </body>
  
  </html>

Para o HTML estes 3 parágrafos estão escritos exatamente da mesma forma. Ele não "enxergou" o monte de espaços entre as palavras no segundo parágrafo e nem as quebras de linha do terceiro.

E o que você verá na página exibida pelo navegador será:

Esta é uma vantagem porque você pode escrever o seu código da maneira que achar mais legível sem que a formatação desse código interfira no que será efetivamente exibido na página do navegador.

São as tags que fazem a formatação do texto que será exibido, e não a forma como você escreve estes textos no seu código fonte HTML.

Mas e se você quisesse escrever realmente um parágrafo com espaçamento maior entre as palavras?

Vovô         viu         a        uva.

Como eu fiz para exibir palavras mais espaçadas uma vez que o HTML não distingue um espaço de dois espaços, de três espaços, de quatros espaços, de n espaços?

Para isso você tem que informar explicitamente cada espaço em branco extra com o código &nbsp;


  <p>
    Vovô &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
    viu &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; a 
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;uva.
  </p>

&nbsp; é o código HTML para o caractere de espaço e o nbsp significa "non break space". Todo caractere especial em HTML começa com um e comercial (&) e termina com um ponto e vírgula (;). Entre estes dois símbolos inserimos o nome do caractere especial que desejamos exibir.

Alguns caracteres especiais
Nome Caractere Código HTML
E comercial & &amp;
Copyright © &copy;
Registrada ® &reg;
Euro € &euro;
Espadas ♠ &spades;

Da mesma forma, uma quebra de linha no texto do seu código fonte não quebra a linha no texto que será exibido pelo navegador. ("quebrar a linha" quer dizer pular para a próxima linha)

É preciso quebrar a linha com a tag <br>.

O código abaixo não produziria o resultado esperado


  <p>
    Primeira linha deste parágrafo.
    Segunda linha deste parágrafo.
    Terceira linha deste parágrafo.
    Quarta linha deste parágrafo.
  </p>

Na verdade o resultado seria:

Nenhuma quebra de linha! Porque o HTML não enxergou as quebras de linha no código fonte.

Agora usando a tag <br>:


  <p>
    Primeira linha deste parágrafo.<br>
    Segunda linha deste parágrafo.<br>
    Terceira linha deste parágrafo.<br>
    Quarta linha deste parágrafo.
  </p>

Resultado:

Ah, agora sim. Dissemos explicitamente ao HTML onde ele deveria quebrar as linhas dentro do parágrafo com a tag <br>.
